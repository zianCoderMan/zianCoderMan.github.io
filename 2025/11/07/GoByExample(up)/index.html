注：Go 语言的自动类型转换

Go 语言的 自动类型转换（隐式类型转换） 规则非常严格，核心原则是：“安全第一，不允许隐式窄化转换，仅支持有限场景的隐式兼容转换”—— 这和 Java 宽松的自动类型转换（比如 int 能隐式转 long、float 能隐式转 double）有很大差异！

1、先明确核心前提

Go 是 静态强类型语言：

  • 静态：变量类型编译时确定，不能动态改变；
  • 强类型：不同类型之间的转换必须显式声明（除非满足 Go 允许的隐式转换规则），禁止 “悄悄转换” 导致的潜在问题（比如精度丢失、溢出）。

对比 Java：Java 是 “静态弱类型倾向”，允许较多隐式转换（比如 int a = 10; long b = a; 合法），而 Go 为了安全，大幅限制了隐式转换的场景。

常见误区

2.变参是切片的 “副本”，但元素是值拷贝

变参 nums 是切片（引用类型），但函数内修改切片本身（比如 nums = append(nums, 5)）不会影响函数外的实参；但修改切片元素（比如 nums[0] = 100）会影响原切片（因为切片共享底层数组）：

切片（包括变参 nums ...int 本质的切片）在底层包含 3 个核心元信息（存在栈上）：

  1. 指针（ptr）：指向存储数据的底层数组（存在堆上）；
  2. 长度（len）：当前切片的元素个数；
  3. 容量（cap）：底层数组的最大容纳量。

当调用 modify(arr...) 时，发生了切片的引用拷贝：

  • 函数内的 nums 切片，会复制原切片 arr 的 3 个元信息（ptr、len、cap）；
  • 此时 nums 和 arr 的 指针完全相同（指向同一个底层数组 [1,2]），但 nums 是独立的 “地址牌 + 标签” 副本（栈上独立存储）。

接下来分析函数内两行代码的差异：

第一行：nums[0] = 100 → 修改 “底层数组的元素”（影响原切片）

  • 执行逻辑：nums[0] 是通过 nums 的指针，找到底层数组的第 0 个位置，把值改成 100；
  • 关键：修改的是 堆上的底层数组（原切片 arr 的指针也指向这个数组）；
  • 结果：原切片 arr 访问底层数组时，第 0 个元素已经变成 100 → 所以原切片受影响。

第二行：nums = append(nums, 5) → 修改 “切片本身的元信息”（不影响原切片）

  • 执行逻辑：

    1. 原 nums 的 len=2、cap=2，append(5) 时容量不足，Go 会创建一个新的底层数组（比如长度 4）；
    2. 把原底层数组的元素 [100,2] 拷贝到新数组，再追加 5 → 新数组变成 [100,2,5]；
    3. 把 nums 的元信息更新：ptr 指向新数组，len=3，cap=4；

  • 关键：修改的是 函数内 nums 切片自身的元信息（指针换了、len/cap 变了），而原切片 arr 的元信息（ptr、len、cap）完全没变化（还是指向旧数组，len=2）；

  • 结果：函数内的 nums 已经指向新数组，但原切片 arr 还指着旧数组 → 原切片不受影响。

  • 影响原切片的情况：通过切片的指针修改底层数组的元素（比如 nums[0] = 100）；

  • 不影响原切片的情况：切片的 len/cap 是自身的元信息，修改后只影响当前切片，不影响其他切片副本。

15.闭包

注：闭包的原理和使用场景

1.函数类型字面量

func() int 是 Go 中的 函数类型字面量，就像 int 是 “整数类型”，func() int 是 “无参返回 int 的函数类型”—— 只不过它描述的是 “函数”，不是 “数据”。

func() int 是 “类型”（类似 int），不是 “具体函数”；“类型” 不需要名字，只有 “具体函数” 才需要（或可以不需要）名字。

举个类比：int 是整数类型（模板），不需要名字；用 int 声明的变量 var a int = 5，a 是具体实例（有名字），而 5 是匿名的实例 —— 函数类型和具体函数的关系完全一样。

举几个函数类型的例子：

函数类型字面量 含义 匹配的函数示例
func() int 无参，返回 int func() int { return 5 }
func(a int) string 1 个 int 参数，返回 string func(x int) string { return strconv.Itoa(x) }
func(a, b int) int 2 个 int 参数，返回 int func(x, y int) int { return x+y }

2.为什么可以返回函数类型？（Go 的 “一等公民” 特性）

Go 中函数是 “一等公民”，意味着：

  1. 函数可以作为 返回值（如示例）；
  2. 函数可以作为 参数（如 func f(oper func(int) int) int）；
  3. 函数可以赋值给 变量（如 var add func(int, int) int = func(x,y int) int { return x+y }）。

3.底层原理（为什么闭包能维护状态？）

Go 中闭包的底层实现是 “函数指针 + 捕获变量的指针”：

  1. 当内部函数引用外部变量时，Go 编译器会将这些变量 “提升” 到堆上（而不是栈上）—— 栈上的变量会随函数执行完毕销毁，堆上的变量会持续存在，直到闭包被销毁；
  2. 闭包实例（比如nextInt）本质是一个 “结构体”，包含：
    • 内部函数的指针（指向要执行的代码）；
    • 捕获变量的指针（指向堆上的 i）；
  3. 每次调用闭包，都会通过指针访问堆上的 i，实现状态的持续维护。

4.闭包的典型使用场景

(1)计数器 / 生成器

比如生成自增 ID、序列值（如示例中的计数器），无需全局变量，用闭包维护状态更安全。

(2) 函数工厂（根据参数生成定制化函数）

(3) 延迟执行（比如回调函数、资源释放）

5.闭包的常见误区

（1）闭包捕获的是变量 “引用”，不是 “值”：

这是 Go 闭包的核心特性（也是坑点）：

  • 闭包捕获外部变量时，捕获的是变量的 内存地址（引用），而不是变量在捕获瞬间的具体值；
  • 循环中只有 一个循环变量 i（内存地址固定），3 个闭包都指向这个地址；
  • 循环结束后，i 的值变成 3，所有闭包通过引用访问到的都是这个最终值。

解决：循环内创建临时变量，捕获临时变量的值（tmp := i; funcs = append(funcs, func() { fmt.Println(tmp) })）。

注：[]func() 是 “切片类型”，切片的元素类型是 “函数类型”—— 可以拆成两部分理解：

  1. 外层 []T：是切片类型（T 是切片的元素类型），比如 []int 是 “int 类型的切片”，[]string 是 “string 类型的切片”；
  2. 内层 func()：是切片的元素类型（函数类型字面量），表示 “无参数、无返回值的函数”（func() 等价于 func() ()）。

所以 []func() 的字面意思是：“一个切片，里面装的每个元素都是「无参无返回值的函数」”—— 就像 []int 装的是 int，[]func() 装的是 “无参无返回值的函数”。

举个类比：

切片类型 元素类型 切片里装的内容
[]int int 多个整数（1、2、3…）
[]string string 多个字符串（”a”、”b”…）
[]func() func() 多个无参无返回值的函数

（2）闭包可能导致内存泄漏

闭包捕获的变量会一直存在于堆上，如果闭包长期不被销毁（比如全局闭包），这些变量也不会被 GC，可能导致内存泄漏 —— 使用时需注意闭包的生命周期。

（3）闭包实例独立

不同闭包实例捕获的变量是独立的，修改一个不会影响另一个（如示例中 nextInt 和 nextInts）。

  • 注意：二进制转换时，必须补够 8 位（比如 0xe0 是 11100000，不是 1110000）。

第三步：按 UTF-8 规则 “提取有效位” 并拼接

根据 3 字节格式，我们只保留所有 x 位置的有效位，然后拼接起来：

  1. 从 0xe0（11100000）中提取有效位：0000（对应格式中的 xxxx）；
  2. 从 0xb8（10111000）中提取有效位：111000（对应格式中的 xxxxxx）；
  3. 从 0xaa（10101010）中提取有效位：101010（对应格式中的 xxxxxx）；
  4. 拼接有效位：0000 + 111000 + 101010 → 完整二进制：0000111000101010。

第四步：将拼接后的二进制转换为十六进制（得到 0x0e2a）

把拼接后的二进制 0000111000101010 转换成十六进制（4 个二进制位 = 1 个十六进制位）：

  • 分组（从右往左，每 4 位一组，左边不足补 0）：0000 1110 0010 1010；
  • 每组转换为十六进制：
    • 0000 → 0；
    • 1110 → e；
    • 0010 → 2；
    • 1010 → a；
  • 拼接结果：0 + e + 2 + a → 0x0e2a（开头的 0 可省略，也能写成 0xe2a，但标准写法会保留 4 位十六进制，即 0x0e2a）。

验证：0x0e2a 确实是 'ส' 的 Unicode 码点

我们可以反向验证：如果知道 'ส' 的 Unicode 码点是 0x0e2a，按 UTF-8 规则编码后，是否能得到 0xe0 0xb8 0xaa？

  1. 0x0e2a 转换为二进制：0000 1110 0010 1010（16 位）；

  2. 按 3 字节格式

    填充：

    • 前 4 位有效位 0000 → 第 1 字节：11100000（0xe0）；
    • 中间 6 位有效位 111000 → 第 2 字节：10111000（0xb8）；
    • 最后 6 位有效位 101010 → 第 3 字节：10101010（0xaa）；

  3. 编码结果和原字节完全一致，证明解码过程正确。

总结：解码的核心逻辑

UTF-8 解码的本质是 “按编码规则提取有效位，拼接后得到 Unicode 码点”，而 rune 类型本质就是 int32，用来存储这个 Unicode 码点 —— 所以 0xe0 0xb8 0xaa 解码后对应的 rune 值就是 0x0e2a。

2.字符串（string）和 rune 的核心区别

对比维度 字符串（string） rune 类型
本质 UTF-8 编码的字节序列（不可变 []byte） 单个 Unicode 字符（等价于 int32）
存储内容 多字节序列（1~4 字节 / 字符） 单个 Unicode 码点（4 字节固定长度）
长度计算 len(s) → 字节数（非字符数） utf8.RuneCountInString(s) → 字符数
遍历方式 普通 for 循环 → 遍历字节（多字节字符会拆分） range 循环 → 自动解码为 rune（正确遍历字符）
适用场景 存储字符串整体、传输数据 处理单个字符（比如字符匹配、字符转换）

3.常见使用场景

1.遍历多字节字符串：永远用 range 循环（自动解码 rune），不要用普通 for 循环（遍历字节会拆分字符）；

2.统计字符数：用 utf8.RuneCountInString(s)，不要用 len(s)（len 是字节数）；

3.字符级操作：比如字符替换、字符匹配、大小写转换，先把字符串转成 []rune（可变的字符切片），操作后再转回 string：

4.字符字面量：单引号 'a'、'中'、'ส' 都是 rune 类型，可直接赋值给 rune 变量

总结：

  • string 存字节，rune 存字符：字符串是 UTF-8 字节的 “容器”，rune 是字节解码后的 “单个字符”；
  • 处理多字节字符靠 rune：遍历、统计、匹配字符时，用 range 或 utf8 包解码成 rune，避免拆分字符。

23.泛型

泛型 = 类型参数 + 约束，解决 “一套代码适配多类型” 的问题，兼顾代码复用和类型安全

泛型的核心语法

Go 泛型的核心语法是「类型参数列表」，放在函数名 / 结构体名后面，格式为 [类型参数 约束, ...]：

语法元素 含义
[K comparable, V any] 类型参数列表：K 和 V 是类型参数，comparable 和 any 是约束；
comparable 约束：表示类型可比较（支持 ==、!=），用于 map 的 key 类型（map 的 key 必须可比较）；
any 约束：等价于 interface{}，表示任意类型（无限制）；
List[T any] 泛型结构体：T 是类型参数，any 约束，结构体的字段 / 方法可使用 T 作为类型；
MapKeys[K, V] 泛型函数：调用时可显式指定类型参数（如 MapKeys[int, string](m)），也可让 Go 自动推断；

注：泛型的核心特性

1. 类型安全：编译时检查类型，避免运行时错误

泛型的类型检查在编译期完成，而非运行期 —— 比如给 List[int] 传入 string 类型，编译时直接报错，比 interface{} 更安全（interface{} 会把类型检查推迟到运行期，容易出现 type assertion 错误）。

2. 类型推断：调用时可省略类型参数（简化写法）

Go 能根据传入的参数自动推断类型参数，无需显式指定：

3. 约束的作用：限制类型参数的范围

约束避免传入不合法的类型，比如 MapKeys 的 K 约束为 comparable，如果传入 map[[]int]string（[]int 不可比较），编译时直接报错：

泛型结构体的方法，接收者必须带类型参数（如 (lst *List[T]) Push(v T)），不能省略 [T]—— 否则方法不属于泛型结构体，编译报错。
