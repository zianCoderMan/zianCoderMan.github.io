49. 字母异位词分组

zianCoderMan
2025-11-05
算法刷题
LeetCode Top100, 哈希

问题与示例：

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

示例 1:

输入: strs = [“eat”, “tea”, “tan”, “ate”, “nat”, “bat”]

输出: [[“bat”],[“nat”,”tan”],[“ate”,”eat”,”tea”]]

解释：

  • 在 strs 中没有字符串可以通过重新排列来形成 "bat"。
  • 字符串 "nat" 和 "tan" 是字母异位词，因为它们可以重新排列以形成彼此。
  • 字符串 "ate" ，"eat" 和 "tea" 是字母异位词，因为它们可以重新排列以形成彼此。

示例 2:

输入: strs = [“”]

输出: [[“”]]

示例 3:

输入: strs = [“a”]

输出: [[“a”]]

题解：

方法一：排序

方法二：计数

上一种解法用「排序」消除顺序差异，而本解法用「计数」直接描述字符特征：对每个字符串，统计 26 个小写字母的出现次数，再将「字母 + 次数」拼接成唯一的 key（比如 “eat” 的 key 是 “a1e1t1”），相同 key 的字符串就是异位词。

补充：

String和StringBuffer对比

对比维度 String StringBuffer
核心特性 不可变字符串 可变字符串（动态修改字符序列）
底层实现 private final char[] value（不可变数组） char[] value（可变数组，支持扩容）
修改操作（拼接 / 替换等） 每次修改都会创建新对象 直接修改原对象，不创建新对象（扩容时内部复制数组，但对外无感知）
效率（修改场景） 低（频繁修改会产生大量临时对象，浪费内存 + GC 开销） 高（无临时对象，直接操作原数组）
效率（读取场景） 高（不可变对象可被缓存，如字符串常量池） 略低（无常量池缓存，读取逻辑和 String 类似，但多了少量额外判断）
线程安全 天然安全（不可变，无修改竞争） 安全（所有方法加 synchronized 锁）
构造方法 / 常用方法 支持直接赋值（String s = "abc"）、构造方法；常用 equals()、hashCode() 等 必须通过构造方法创建（new StringBuffer()）；常用 append()、insert()、delete() 等
内存占用（修改场景） 高（临时对象堆积） 低（仅维护一个可变数组）

StringBuilder和 StringBuffer 功能几乎完全一致（都是可变字符串），唯一区别是 线程安全：

  • StringBuffer：所有方法加 synchronized，线程安全，但效率略低；
  • StringBuilder：无锁，线程不安全，但效率更高（JDK 1.5+ 新增）。
